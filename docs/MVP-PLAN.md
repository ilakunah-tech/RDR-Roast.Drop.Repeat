# План разработки MVP — Roasting Desktop App

Цель: десктопное приложение для обжарки кофе с UI по образцу Cropster Roasting Intelligence, симулятором, подключением к Besca и Diedrich, сохранением в .alog.

---

## 1. Обзор MVP

### 1.1 Экраны

| Экран | Назначение |
|-------|------------|
| **Главный** | График (BT, ET, RoR BT, RoR ET), события, фазы, текущие значения, Duration, Abort/Stop. |
| **Настройки** | Источник данных (симулятор / Besca / Diedrich), параметры подключения, единицы °C/°F, путь сохранения. |

### 1.2 Кривые на графике

- **BT** (Bean Temperature)  
- **ET** (Exhaust Temperature)  
- **RoR BT** (Rate of Rise по BT)  
- **RoR ET** (Rate of Rise по ET)  

RoR считаются в домене по выбранному окну (например 30–60 с) и сглаживанию.

### 1.3 События

- **Charge** — старт (загрузка зелени)  
- **TP** (Turning point)  
- **CC** (Color change / dry end)  
- **Drop** — конец обжарки  

Отображаются вертикальными линиями/маркерами и подписями на графике.

### 1.4 Фазы (три)

- **Drying** (сушка)  
- **Maillard** (реакция Майяра)  
- **Development** (развитие после первого крэка)  

На графике — горизонтальные полоски (как на референсном скрине). В каждой полоске: **название фазы**, **длительность (MM:SS)**, **доля от общего времени (%)**. Таймеры фаз встроены в эти полоски, отдельный блок «Maillard / Duration» внизу не дублируем — только Duration в нижней панели.

### 1.5 Текущие значения (правая панель)

- **Bean temp** (°C или °F)  
- **Exhaust temp**  
- **RoR BT**  
- **RoR ET**  

Обновляются в реальном времени во время записи.

### 1.6 Управление

- **Abort** — прервать без сохранения.  
- **Stop** — остановить и сохранить профиль в .alog.  
- **Duration** — общая длительность обжарки (внизу).  
- Название обжарки / лот в MVP не вводим (можно автоимя по дате при сохранении).

### 1.7 Что не в MVP

- Reference (Duration, Dev time, Agtron и т.д.) — позже.  
- Comments (список событий с галочками) — позже.  
- Предсказание первого крэка (Smart predictions) — позже.  
- Список сохранённых обжарок — позже.

## 2. Источники данных

### 2.1 Симулятор

- Генерирует BT и ET по времени (например, типовые кривые обжарки).  
- RoR BT и RoR ET считаются в приложении по этим кривым (доменная логика).  
- Запуск/остановка симуляции через те же кнопки (Start/Stop), источник выбирается в настройках.

### 2.2 Ростеры Besca и Diedrich

- Реализовать **два драйвера** (адаптера к общему интерфейсу `RoastDataSource`).  
- **Настройки** взять и адаптировать из Artisan:  
  - **Besca**: `src/includes/Machines/Besca/*.aset` (MODBUS: comport, host, регистры input1/input2 для BT/ET и т.д.).  
  - **Diedrich**: `src/includes/Machines/Diedrich/*.aset` (конфигурация устройств, порты, каналы).  
- В настройках: выбор машины (Besca / Diedrich), порт, адрес, при необходимости маппинг регистров/каналов. Сохранение в конфиг приложения (не в .alog).

### 2.3 Общий контракт

- Интерфейс `RoastDataSource`: поток сэмплов (время, BT, ET), состояние подключения, connect/disconnect с таймаутом.  
- UI и домен не зависят от конкретного протокола (MODBUS/Serial и т.д.).

---

## 3. Сохранение профиля (.alog)

- Формат **.alog** — по примеру Artisan: сериализованный словарь (как в Artisan `getProfile()` + `serialize()` в `artisanlib/util.py`).  
- Минимальный совместимый набор полей для MVP:  
  - `timex` — массив времени (с),  
  - `temp1` — BT, `temp2` — ET,  
  - `timeindex` — индексы событий (Charge, TP, CC, Drop),  
  - `specialevents`, `specialeventstype`, `specialeventsvalue`, `specialeventsStrings` — события,  
  - `phases` — границы фаз (или длительности),  
  - `mode` — 'C' или 'F',  
  - при необходимости `roastdate`, `roasttime`, `version` для совместимости.  
- Сохранение вызывается по **Stop**; путь к файлу — из настроек (или диалог «Сохранить как»). Имя файла можно формировать по дате/времени, без ввода названия обжарки в MVP.

---

## 4. Экран настроек (MVP)

- **Источник данных**: Симулятор | Besca | Diedrich.  
- **Для Besca / Diedrich**: порт (COM/serial), адрес/ID, при необходимости путь к .aset или ручной ввод ключевых параметров (адаптированных из Artisan).  
- **Единицы**: °C / °F.  
- **Путь сохранения** по умолчанию для .alog.  
- Сохранение настроек в локальный конфиг (например QSettings или JSON в папке приложения).

## 5. Этапы разработки (порядок работ)

### Фаза 0: Каркас проекта

- Gradle (Kotlin DSL), JDK 17/21, модуль приложения.  
- Зависимости: JavaFX, ControlSFX, j2mod, jSerialComm, логирование.  
- Точка входа, главное окно (пустой layout по скрину: график по центру, правая панель, низ).

### Фаза 1: Домен и симулятор

- Домен: `RoastProfile` (timex, temp1, temp2), события (Charge, TP, CC, Drop), расчёт RoR (BT, ET), три фазы (Drying, Maillard, Development) и их длительности/проценты.  
- Симулятор: генерация BT/ET по времени, выдаёт сэмплы в формате общего интерфейса.  
- Юнит-тесты на расчёт RoR и фаз.

### Фаза 2: Главный экран (UI по скрину)

- График: четыре кривые (BT, ET, RoR BT, RoR ET), две оси Y (температура и RoR).  
- События: вертикальные линии/маркеры + подписи (Charge, TP, CC, Drop).  
- Фазы: три горизонтальные полоски с названием, длительностью и процентом.  
- Правая панель: текущие значения (BT, ET, RoR BT, RoR ET).  
- Низ: Duration, кнопки Abort и Stop.  
- Подключение к симулятору: Start → поток данных → обновление графика и значений на FX thread.

### Фаза 3: Запись и сохранение

- По Start — начало записи в буфер (доменная модель профиля).  
- По Stop — остановка, диалог сохранения или путь из настроек, запись .alog (совместимая структура с Artisan).  
- По Abort — сброс без сохранения.

### Фаза 4: Драйверы Besca и Diedrich

- Изучить конфиги Artisan для Besca и Diedrich (.aset, MODBUS/Serial).  
- Реализовать два адаптера к `RoastDataSource`: чтение BT/ET по протоколу, таймауты, переподключение.  
- В настройках: выбор машины, порт, адрес (и при необходимости маппинг регистров). Сохранение/загрузка конфига.

### Фаза 5: Настройки и полировка

- Экран настроек: источник (Симулятор / Besca / Diedrich), параметры подключения, °C/°F, путь сохранения.  
- Применение настроек при запуске и при смене источника.  
- Проверка полного цикла: симулятор → запись → Stop → .alog; затем Besca/Diedrich при наличии железа.

## 6. Критерии готовности MVP

- [ ] Запуск приложения, главное окно в стиле скрина (график, панели, кнопки).  
- [ ] Симулятор: запись BT/ET, отображение четырёх кривых и RoR в реальном времени.  
- [ ] События Charge, TP, CC, Drop (ручная расстановка или по правилам симулятора) и отображение на графике.  
- [ ] Три фазы на графике с длительностью и процентом.  
- [ ] Текущие значения BT, ET, RoR BT, RoR ET; Duration; Abort и Stop.  
- [ ] Сохранение по Stop в .alog (формат, совместимый с Artisan).  
- [ ] Настройки: источник, Besca/Diedrich (порт, адрес), единицы, путь сохранения.  
- [ ] Подключение к Besca и Diedrich (адаптеры с конфигом из Artisan), запись с реального ростерa.
