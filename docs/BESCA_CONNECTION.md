# Подключение RDR к ростеру Besca

## TCP (Ethernet)

1. **Сеть**: ПК и ростер должны быть в одной сети (один Wi‑Fi или один коммутатор).
2. **IP ростера**: Узнайте IP в меню ростера или у поставщика. Часто `10.0.0.9` или `192.168.1.x`.
3. **Проверка сети** (в cmd):
   ```bat
   ping 10.0.0.9
   ```
   Если пинг не проходит — проверьте кабель, Wi‑Fi, что ростер включён и что IP верный.
4. **Порт**: Modbus TCP обычно 502. В настройках RDR: Подключение → Host (TCP) = IP ростера, Port (TCP) = 502.
5. **Брандмауэр**: Разрешите исходящие подключения на порт 502 для RDR (или временно отключите для проверки).

## Если «Connect timed out»

- Ростер не отвечает по TCP в течение 15 секунд. Возможные причины:
  - неверный IP (проверьте пинг);
  - ростер выключен или Modbus TCP не включён в меню;
  - другой порт (не 502) — уточните в инструкции Besca;
  - брандмауэр или антивирус блокирует порт 502.

## Serial (COM)

Если TCP недоступен, можно использовать Modbus RTU по USB‑адаптеру (COM‑порт). В RDR выберите транспорт «Serial (COM port)», укажите COM‑порт и скорость (часто 9600), Slave ID = 1.
